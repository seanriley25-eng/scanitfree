## scanitfree Resume Reviewer — Summary

The scanitfree Resume Reviewer is a **web-based, paste-text, no-signup AI tool** that evaluates a plain-text resume and returns an A–F letter grade, an overall score (0–100), a separate ATS compatibility score (0–100), per-section breakdowns with individual progress-bar scores, a "Top Issues to Fix" list, and a 2–3 sentence summary. It is powered by Claude Sonnet 4, free at 15 reviews/day per IP, and requires no account.

**Prompt focus:** Formatting/readability, quantified achievements, ATS keyword optimization, section completeness, grammar/consistency, and overall positioning — evaluated in a single pass with no job-description context and no role/industry differentiation.

## What scanitfree Has That Competitors Don't

1. **Text paste as primary input — desktop-friendly without file handling** — Every competitor requires a PDF or DOCX upload. scanitfree's textarea lets users paste from any source (Word, Google Docs, LinkedIn's "Copy" export, a plain .txt file) with zero format friction. This is a meaningful UX advantage for users who don't have a clean PDF ready.

2. **A–F letter grade for holistic resume quality** — No competitor applies an A–F letter grade to resume *quality* (Jobscan uses a match %, Kickresume uses raw numbers, Enhancv uses a number). The grade is instantly legible ("I'm a C, I need to improve") in a way a 67/100 isn't. It also maps cleanly to the food-safety tool's visual language — consistent brand UX across scanitfree tools.

3. **Dual-score output in a single pass: overall + ATS** — Most tools give either an overall quality score or an ATS score. scanitfree surfaces both simultaneously without requiring the user to paste a job description. This is architecturally efficient and gives more information density than competitors offering only one score.

4. **15 free scans/day per IP, no account ever** — Jobscan's free tier is 5 scans/month (not per day — per month). ApplyBuddy is truly no-account but limits tailored resumes. Enhancv is free but emails results. scanitfree offers the most generous frictionless access in the entire competitive set.

5. **Powered by Claude Sonnet 4** — All competitors use GPT-4/ChatGPT. This is a minor differentiator today but a real one: Claude's longer context window handles verbose senior-level resumes without truncation issues, and Claude's instruction-following produces cleaner structured JSON. Marketable as "not ChatGPT" for users distrustful of OpenAI.

## Recommended Additions

Prioritized by SEO + conversion impact, competitive gap size, and paid-tier monetization potential:

---

### 🥇 #1 — Job Description Matching (JD Paste)
**Rationale — highest SEO + conversion lift in the field:** The most-searched modifier across all resume tool keywords is "for [job title]" or "with job description." Queries like "ATS resume checker for software engineer" and "resume scanner match job description" have massive volume and very high intent — these users are applying right now. Adding a second optional textarea ("Paste the job description you're applying to — optional") unlocks role-specific keyword gap output without requiring any infrastructure beyond updating the Claude prompt. Jobscan has built a $50/month business on this single feature. scanitfree can offer the basic version free (keyword overlap + match %) and gate deeper features (auto-insert suggestions) on paid.

**Implementation path:** Add optional `jobDescription` field to the API body. When provided, expand the prompt: "Also compare the resume against this job description. Add `keywordGaps: ["<missing keyword>"]` and `matchScore: <0-100>` to the JSON output. Flag the top 5 missing keywords as 'warning' issues." When absent, current behavior unchanged — zero regression.

---

### 🥈 #2 — PDF Upload (via Claude Vision)
**Rationale — table stakes gap:** Being the only tool in the space that doesn't accept PDF is a significant trust and conversion barrier. Users arrive with a PDF; asking them to copy-paste all text first creates friction and drop-off. The scanitfree stack already calls Claude — Claude's vision API can read a PDF natively. This is a one-endpoint change: detect file upload, pass to Claude vision for text extraction, then pipe extracted text into the existing analysis prompt.

**Implementation path:** Add a file input alongside the textarea in `client.tsx`. On the API side in `route.ts`, if a file is attached, send it to Claude as a `document` content block (Claude supports PDF natively via the Messages API). Extract text in a first pass if needed, or pass the document directly and let Claude analyze it inline. Output is identical to current text-paste path.

---

### 🥉 #3 — AI Bullet Point Rewriter (per Top Issue)
**Rationale — most defensible paid-tier feature:** ResumeWorded, ApplyBuddy, SkillSyncer, and Kickresume all offer AI rewrites — and competitors charge for it. Users care deeply about "tell me the exact words to use" rather than "tell me what's wrong." The current output identifies issues but leaves rewriting to the user. Generating 1–2 rewritten alternatives per flagged bullet (displayed as "Before → After") dramatically increases perceived value and time-on-page. This is the clearest paid-tier candidate in the resume niche: give the diagnosis free, charge for the prescription.

**Implementation path:** Expand the JSON schema to add `"rewrite": "<improved version>|null"` to each `topIssues` entry (or a new `"rewrites": [{"original": "...", "improved": "..."}]` array). Gate this array's display behind a paid account flag — free users see the issue label and a "Unlock rewrite →" CTA, paid users see the full before/after comparison inline.

---

### 🏅 #4 — Keyword Gap Analysis (standalone, no JD required)
**Rationale — high-trust signal, zero-friction addition:** Even without a job description, the AI can identify whether the resume is missing expected section-level keywords for the apparent role (inferred from job titles, company names, tech stack, etc. within the resume itself). Output: "Your resume appears to target software engineering roles. Missing keywords commonly expected by ATS for this role: Docker, CI/CD, system design, cross-functional." This can be surfaced as a `"keywordGaps"` array in the existing JSON schema with no user-input change required. It also creates a natural upgrade hook: "Paste a job description to get a personalized match score → [upgrade button]."

**Implementation path:** Add `"inferredRole": "<string>"` and `"keywordGaps": ["<keyword>"]` to the prompt output schema. Instruct the model: "Infer the user's target role from the resume content and list 5–8 high-value ATS keywords commonly expected for that role that are missing from this resume." Render as a new "Missing Keywords" section with pill-style badges.

---

### 🎖️ #5 — Named ATS System Transparency (e.g., "Workday-style check")
**Rationale — trust and SEO differentiation:** KudosWall and Jobscan (paid) explicitly name the ATS systems they simulate (Workday, Taleo, Greenhouse). This signals depth and specificity that generic "ATS compatible" claims don't. Users searching "will my resume pass Workday ATS" need a specific answer. scanitfree's current `atsScore` is an AI estimate with no named methodology. Adding a `"atsNotes"` field with 2–3 specific system-style observations ("Workday-style parsers reject multi-column layouts; your resume uses single-column — ✓", "Taleo truncates summaries over 150 words — your summary is 210 words — ✗") would make the ATS score feel credible rather than arbitrary.

**Implementation path:** Expand the prompt: "For the `atsScore`, also output `atsNotes: [{"system": "Workday|Taleo|Greenhouse", "check": "<what was checked>", "pass": true|false, "note": "<specific observation>"}]` covering at least 2 common enterprise ATS systems." Render as an "ATS System Checks" expandable section below the main ATS score.
