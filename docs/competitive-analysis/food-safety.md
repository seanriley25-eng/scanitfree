## scanitfree Food Safety Scanner — Summary

The scanitfree Food Safety Scanner is a **web-based, no-signup AI tool** that accepts a product name, ingredient list, or UPC code as plain text and returns an A–F letter grade (0–100 numeric score), severity-flagged concerns (danger / warning / info), and a 2–3 sentence narrative summary. It is powered by Claude Sonnet 4 and is free to use at 15 scans/day per IP with no account required.

## What scanitfree Has That Competitors Don't

1. **Web-first + no app download, no signup, no credits** — Most ingredient AI tools are apps (Yuka, Trash Panda, Bobby Approved, Ingredient AI) or credit-gated (Ingredient Checker). scanitfree is instant-use in any browser with no friction whatsoever and 15 free scans/day.

2. **A–F letter grade system for food safety specifically** — Fooducate and Open Food Facts use A–E/A–F for *nutrition*, not safety. No competitor directly maps food *safety* risk to an A–F letter grade; scanitfree's grade is explicitly safety-oriented (FDA enforcement actions, recalls, allergen labeling risk).

3. **Three-tier severity flagging on a per-concern basis** — The danger/warning/info tiering applied per individual concern (not just an aggregate score) is more actionable than most competitors' binary red/green or single-score outputs.

4. **UPC/barcode input without a camera** — Users can type a UPC code into scanitfree without needing to point a phone camera. This serves desktop users (e.g., typing in a barcode number from a package) which no mobile-first competitor supports.

5. **FDA enforcement action + recall cross-referencing in the AI prompt** — The system prompt explicitly instructs the model to cross-reference FDA enforcement actions and recent recalls, giving some recall context even without a live database feed.

## Recommended Additions

Prioritized by impact-to-effort ratio, competitive differentiation, and alignment with "food safety" positioning (vs. nutrition tracking):

---

### 🥇 #1 — Live FDA Recall Feed Integration
**Rationale:** This is scanitfree's biggest credibility gap. The AI prompt *mentions* recalls, but the model's knowledge has a training cutoff — it cannot know about a recall issued last week. Every competitor in the "recall checker" keyword space (Recalls.gov, FoodSafety.gov, FoodReady) uses live data. Integrating the FDA's public recall RSS feed or OpenFDA API at query time would make scanitfree the **only free web tool that combines AI ingredient analysis with live recall checks** — a uniquely defensible position. The OpenFDA API is free and requires no key for basic use.

**Implementation path:** On submit, query `api.fda.gov/food/enforcement.json?search=…` with the product name/UPC, append any matching recall records to the Claude prompt as grounding context. Flag with a distinct "⚠️ ACTIVE RECALL" severity block in the output.

---

### 🥈 #2 — Photo / Label OCR Upload
**Rationale:** Trash Panda, Scan AI, and ingredientschecker.in all accept photos. For users at the grocery store or with physical packaging, text entry is a barrier. A photo upload that runs OCR (via Claude's vision API or a lightweight OCR pass) and auto-populates the ingredient field would dramatically improve mobile conversion. This also opens "snap a photo" as a differentiating CTA for social sharing.

**Implementation path:** Add a file input alongside the textarea; pass the image to Claude's vision endpoint to extract the ingredient list before the safety analysis pass. Two-step prompt: (1) extract text → (2) analyze safety.

---

### 🥉 #3 — Per-Flag Source Citations (FDA / Research Links)
**Rationale:** Trash Panda's biggest UX differentiator is per-ingredient peer-reviewed citations. Users distrustful of AI ("why should I believe this?") need evidence. scanitfree's current output gives conclusions with no backing. Adding 1–2 source URLs per danger/warning flag (FDA page, PubMed study, CSPI database) would dramatically increase trust and time-on-page. The AI prompt can be expanded to output `"source_url"` alongside each flag's text.

**Implementation path:** Expand the JSON schema in `TOOL_PROMPTS["food-safety"]` to include `"source_url": "<string|null>"` per flag. Instruct the model to return the most relevant FDA enforcement page or CSPI/EWG URL when available. Render as a "Learn more →" link under each flag.

---

### 🏅 #4 — Ultra-Processing Level (NOVA Classification)
**Rationale:** Open Food Facts is the only competitor offering NOVA scoring, and it's a rapidly growing consumer health metric (driven by the ultra-processed food research wave). Adding a NOVA group (1–4) to the output JSON gives scanitfree a differentiated data point that even Yuka and Trash Panda lack. It's a free-tier addition that reinforces the safety-first brand while adding nutritional intelligence.

**Implementation path:** Add `"nova_group": <1|2|3|4>` and `"nova_label": "<Unprocessed|Processed culinary|Processed|Ultra-processed>"` to the prompt output schema. The AI can infer NOVA class from ingredient list characteristics (presence of additives, hydrolyzed proteins, industrial oils, etc.).

---

### 🎖️ #5 — Dietary Filter / Allergen Profile (Logged-In Feature)
**Rationale:** Yuka and Trash Panda both gate allergen customization behind a paid tier. scanitfree could offer it free with account creation — a low-cost conversion to registered users. Letting users set a profile (e.g., "I'm allergic to tree nuts and avoid gluten") means each scan auto-elevates relevant flag severity and adds a personalized "⚠️ CONTAINS YOUR ALLERGEN" banner. This directly competes with Yuka's $15/year tier on a feature users care deeply about.

**Implementation path:** Add optional user auth (NextAuth.js or similar). Store `allergen_profile[]` and `dietary_flags[]` per user. On analyze, pass the profile into the Claude prompt: "User is allergic to: [tree nuts]. Elevate any flag mentioning these to 'danger' severity and add a personal allergen warning block."
